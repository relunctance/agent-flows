# Agent Flows 架构文档

## 概述

Agent Flows 是一个多阶段工作流程定义与执行框架，支持不同类型的团队（软件研发、律师咨询、股票分析等）定义和执行流程。

## 核心概念

### Flow（流程）

Flow 是完整的工作过程定义，包含多个有序的阶段（Phase）。

### Phase（阶段）

阶段是流程中的一个工作单元，有明确的责任人和可定义的评审机制。

## 架构设计

```
┌─────────────────────────────────────────────────────────────┐
│                        Expert Teams                          │
│                   (团队管理 + 命令行界面)                     │
└─────────────────────────┬───────────────────────────────────┘
                          │ 调用
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                      Agent Flows CLI                         │
│              python -m agent_flows.cli <cmd>                │
└─────────────────────────┬───────────────────────────────────┘
                          │ 读取
                          ▼
┌─────────────────────────────────────────────────────────────┐
│                      Flows 定义                             │
│                 flows/*.toml                                │
│  - software-dev.toml (软件研发流程)                         │
│  - legal-consultation.toml (律师咨询流程) [规划中]          │
│  - stock-analysis.toml (股票分析流程) [规划中]              │
└─────────────────────────────────────────────────────────────┘
```

## CLI 命令设计

| 命令 | 功能 | 输入 | 输出 |
|------|------|------|------|
| `list` | 列出所有可用 flow | 无 | flow 名称列表 |
| `load` | 绑定 flow 到团队 | flow 名, team_id | 成功/失败 |
| `status` | 获取团队当前阶段 | team_id | 当前阶段信息 |
